# <a name="set-up-a-microsoft-365-developer-sandbox-subscription"></a>Настройка подписки на песочницу для разработчиков Microsoft 365 

Настройте песочницу Microsoft 365 для разработчиков, чтобы создавать решения независимо от рабочей среды. Членство в программе для разработчиков, которое может быть получено обычными участниками или через Visual Studio Enterprise, включает подписку на Microsoft 365 E5 для разработчиков с лицензиями на 25 пользователей. Она действует в течение 90 дней и бесплатна для использования только в целях разработки (решения для написания кода). 

> [!NOTE] 
> Чтобы настроить подписку, необходимо сначала [присоединиться к программе для разработчиков Microsoft 365](microsoft-365-developer-program.md) непосредственно или через Visual Studio Professional или Enterprise (если у вас есть подписка). После присоединения вы увидите возможность настройки подписки.

Вы можете настроить либо готовую, либо настраиваемую песочницу.

## <a name="instant-sandbox"></a>Готовая песочница

При выборе готовой песочницы вы можете сократить время настройки песочницы с нескольких дней до секунд. В мгновенной песочнице Microsoft 365 предварительно подготовлены Microsoft Teams, SharePoint, Outlook и Office. Кроме того, она содержит лицензии для 24 тестовых пользователей и одного администратора, а также новейшие приложения Microsoft 365 для ПК и Mac, включая Power Apps, Power BI, Enterprise Mobility + Security, Office 365 Advanced Threat Protection и Azure Active Directory.

В дополнение вы получаете предварительно установленные примеры данных, в том числе новый пакет примеров данных Teams и примеры данных пользователей, почты и календаря для Microsoft Graph и SharePoint Framework, имитирующие корпоративную среду небольшой компании для создания решений на платформе Microsoft 365. Подробнее см. в статье "Примеры данных песочницы для разработчиков".

> [!VIDEO https://www.microsoft.com/en-us/videoplayer/embed/RWOmCY]

Выбрав этот вариант, вы не сможете настроить доменное имя.

## <a name="configurable-sandbox"></a>Настраиваемая песочница

Если вы выберете настраиваемую песочницу, у вас будет возможность настроить доменное имя. Вам будет предоставлена пустая песочница, которую необходимо заполнить примерами данных. Подготовка этой песочницы может занять до двух дней. Вы можете устанавливать по одному пакеты примеров данных, доступные на вашей панели мониторинга программы для разработчиков. Подробнее см. в статье "Примеры данных песочницы для разработчиков".

## <a name="set-up-your-microsoft-365-e5-sandbox-subscription"></a>Настройте подписку Microsoft 365 E5 на песочницу для разработчиков

Чтобы получить подписку на Microsoft 365 для разработчиков, выполните следующие действия.

1. На странице профиля щелкните **Настройка подписки на E5**.

2. В диалоговом окне **Настройка подписки на Microsoft 365 E5 для разработчиков** выберите готовую или настраиваемую песочницу, а затем нажмите кнопку **Далее**.

    ![Снимок экрана диалогового окна "Настройка подписки на Microsoft 365 E5 для разработчиков"](images/set-up-your-subscription.png)

Если вы выбрали готовую песочницу, выполните следующие действия.

1.  Выберите **страну или регион для вашего центра обработки данных**, затем введите **имя пользователя для администратора** и **пароль администратора**, а также альтернативный пароль для ваших вымышленных пользователей (необязательно), после чего нажмите **Продолжить**.

    > [!IMPORTANT] 
    > Запишите имя пользователя и пароль, поскольку они потребуются для доступа к подписке разработчика.

2.  Введите действительный номер телефона и щелкните **Отправить код**. Введите полученный код и нажмите **Настроить**.

    > [!NOTE] 
    > Необходимо использовать действительный номер мобильного телефона, а не протокол VoIP.

3.  После создания подписки ее доменное имя и срок действия отобразятся у вас на странице профиля.

Если вы выбрали настраиваемую песочницу, выполните следующие действия.

1.  Выберите **страну или регион**, а затем введите имя пользователя в поле **Создание имени пользователя** и доменное имя в поле **Создание доменного имени**. Создайте и подтвердите пароль, а затем нажмите **Продолжить**.

    > [!IMPORTANT] 
    > Запишите имя пользователя и пароль, поскольку они потребуются для доступа к подписке разработчика.

2.  Введите действительный номер телефона и щелкните **Отправить код**. Введите полученный код и нажмите **Настроить**.
3.  После создания подписки ее доменное имя и срок действия отобразятся у вас на странице профиля.


## <a name="configure-the-subscription"></a>Настройка подписки

1. На странице профиля выберите команду **Перейти к подписке** и выполните вход с помощью своего идентификатора пользователя (например, username@domain.onmicrosoft.com) и пароля, указанных для подписки разработчика.

   > [!NOTE] 
   > Не входите в подписку с использованием идентификатора учетной записи программы для разработчиков.

2. Используйте средство запуска приложений, чтобы перейти в [Центр администрирования](https://admin.microsoft.com/AdminPortal/Home#/homepage).

3. На домашней странице Центра администрирования нажмите кнопку **Перейти к пошаговой настройке**. Откроется страница **Установка Microsoft 365 E5 для разработчиков**.

4. **Установите приложения Office**. Вы можете установить приложения Office на свой компьютер. Когда все будет готово, щелкните **Продолжить**.

5. **Настройте персональные данные входа и электронной почты**. Вы можете подключить свою подписку к домену или использовать существующий поддомен, который вы создали. Когда все будет готово, щелкните **Использовать этот домен** или **Позже**.

6. **Добавьте новых пользователей**. Вы можете добавить вымышленных или реальных пользователей для помощи в разработке. Когда все будет готово, щелкните **Добавить пользователей и назначить лицензии**.
    
    > [!NOTE]
    > Если у вас настраиваемая подписка, после ее настройки вы можете установить пакет примеров данных "Пользователи". Пример пакета данных "Пользователи" создает в вашей подписке 16 вымышленных пользователей и добавляет лицензии, а также почтовые ящики, имена, метаданные и фотографии для каждого пользователя. Подробнее см. в статье [Примеры данных песочницы для разработчиков](install-sample-packs.md).

6. **Назначьте лицензии нелицензированным пользователям**. Предоставьте лицензии всем пользователям, которым нужна возможность работы с этой подпиской. Когда все будет готово, щелкните **Добавить пользователей и назначить лицензии** или **Позже**.

7. **Отправьте учетные данные для входа**. Всем настоящим пользователям с доступом к подписке нужно отправить их учетные данные для входа. Можно выбрать способ отправки, например электронную почту, скачивание или печать. Когда все будет готово, щелкните **Продолжить**.

   > [!TIP] 
   > Впоследствии каждый раз перед тем, как перейти на панель мониторинга, следует выполнить вход с помощью своей учетной записи *username@domain*.onmicrosoft.com.

8. Выберите, хотите ли вы отправить пользователям письмо о Microsoft Teams, и нажмите **Продолжить**.

9. **Вы достигли завершения настройки**. Вы завершили настройку своей подписки. Вы можете оценить процесс при желании. После завершения **перейдите в Центр администрирования**.
    
   > [!NOTE] 
   > В настоящее время регионом по умолчанию для подписки является Северная Америка, независимо от страны или региона, в котором вы находитесь. Вы можете продолжать настройку и использование подписки разработчика.

## <a name="provision-microsoft-365-services-in-your-configurable-sandbox"></a>Подготовка служб Microsoft 365 в настраиваемой песочнице

Если у вас настраиваемая песочница, то внутренним службам, таким как SharePoint и Exchange, может потребоваться некоторое время на подготовку для подписки. На этом этапе некоторые значки в средстве запуска приложений и на домашней странице отображаются как **Настройка (это приложение по-прежнему настраивается)**. Это займет не более часа.

После завершения подготовки новую подписку на Microsoft 365 можно использовать для разработки. Срок действия подписки истекает через 90 дней. Чтобы продлить его, см. раздел [Когда срок действия моей подписки будет истекать, смогу ли я его продлить?](microsoft-365-developer-program-faq.yml#renew-subscription)

Также рекомендуется включить варианты получения выпусков, чтобы обеспечить скорейший доступ к новейшим возможностям Microsoft 365. Дополнительные сведения см. в статье [Настройка вариантов стандартного или целевого получения выпусков](https://support.office.com/article/set-up-the-standard-or-targeted-release-options-in-office-365-3b3adfa4-1777-4ff0-b606-fb8732101f47).

## <a name="set-up-a-microsoft-azure-account"></a>Настройка учетной записи Microsoft Azure

Для некоторых решений Office может потребоваться учетная запись Microsoft Azure, чтобы обеспечивать создание с помощью служб Azure. Она не входит в подписку на Microsoft 365 для разработчиков. Чтобы настроить бесплатную учетную запись Azure, см. раздел [Создайте бесплатную учетную запись Azure уже сегодня](https://azure.microsoft.com/free/).

## <a name="install-sample-data-packs"></a>Установка примеров пакетов данных

В своей настраиваемой песочнице вы можете установить пакеты примеров данных. Если вы выбрали готовую песочницу, пакеты с примерами данных будут установлены в ней заранее.

Примеры пакетов данных позволяют экономить время, автоматически устанавливая данные и содержимое, необходимые для создания и тестирования решений. Сюда относятся вымышленные пользователи, метаданные и фотографии для имитации небольшой корпоративной среды. Подробнее о доступных пакетах примеров данных и порядке их установки см. статью [Примеры данных песочницы для разработчиков](install-sample-packs.md).
